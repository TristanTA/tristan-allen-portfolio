# LifeNotes

Note capture and retrieval system exploring how semantic search, transcription, and structured workflows can help people reuse what they learn.

## Problem

Most note-taking tools are good at storing information and weak at helping people find, connect, and apply it later. LifeNotes is built around a practical question: how do you turn notes into something more useful than an archive?

## System Built

LifeNotes is an applied AI product concept focused on note intelligence. The system combines structured storage, retrieval-oriented design, and planned model-assisted processing so that captured information becomes easier to search, revisit, and act on.

## Stack and Architecture

- Streamlit-based prototype interface
- Structured note storage
- Early semantic retrieval workflow
- Planned transcription pipeline for voice capture
- Planned extraction and organization layer for turning raw notes into more useful units

## Constraints

- Keep capture friction low
- Preserve privacy and local control where possible
- Avoid adding AI features that do not improve actual retrieval
- Build toward usefulness incrementally instead of overdesigning the first version

## What Works Today

- Note capture interface
- Structured storage for captured entries
- Early semantic search experimentation
- Product framing that connects technical work to a clear user need

## Evidence and Current Status

LifeNotes is in active development. The strongest current evidence is the implemented capture workflow, the retrieval direction, and the clear path from note storage to richer semantic workflows.

Several higher-level features are still in progress or planned, including transcription, stronger retrieval quality, and idea extraction. They are included in the roadmap because they directly shape the architecture, but they are not presented here as completed capabilities.

## Why It Matters

LifeNotes shows how I approach applied AI beyond model novelty. The interesting part is not simply adding an LLM to notes. It is building a system where retrieval, interface decisions, and model-assisted structure all support a better workflow for the user.

This is also where my behavioral science background is useful: it helps me focus on whether the system changes what a person can actually do with information, not only whether the model produces plausible output.

## Next Engineering Milestone

- Improve semantic retrieval quality and indexing behavior
- Add the transcription layer for multimodal capture
- Begin structured idea extraction and linking workflows
